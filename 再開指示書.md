## 2. 本日対応した修正一覧

### ① Gemini テキスト生成が毎回リトライになる問題
- **原因**: `system_instruction` にエロ系キャラクタープロンプトを入れると、モデルのRLHF訓練が内部的に出力を拒否（`finishReason: STOP` + 空レスポンス）
- **修正**: `system_instruction` はゲームロジックのみにし、キャラクタープロンプトをユーザーメッセージ冒頭に移動

### ② Gemini TTS で声が変わる問題
- **原因**: Gemini TTS の `safetySettings` が未設定で、高欲情度のセリフがブロックされGCP TTSにフォールバック
- **修正**: `speakWithGemini` に `safetySettings: BLOCK_NONE` を追加。TTSプロンプトから「欲情度○%」などの過激表現を除去

### ③ Gemini TTS クォータ超過後も試行し続ける問題
- **原因**: 1日100リクエスト上限を超えても毎ターン Gemini TTS を呼び続けて遅延が発生
- **修正**: 429エラー検知時に `geminiTtsQuotaRef.current = true` をセットし、セッション内で以降の呼び出しをスキップ

### ④ しりとりにならない語句が通過する問題
- **修正①**: システムプロンプトのバリデーションルールを順序付き明示条件に強化
- **修正②**: ひらがな・カタカナ入力時はクライアント側で先頭文字・「ん」終わりを事前チェック

### ⑤ ホームボタン・テキスト入力のUI不具合
- ホームボタン押下時に `arousal`・`history`・`gameResult` 等を完全リセット
- テキスト入力のEnter送信時に前回のAI返答テキストをクリア（音声入力と同じ挙動に統一）

### ⑥ 続きから再開機能の削除
- ゲームの性質上不要なため、`handleResumeGame`・`saveGameProgress`・`clearSaveData`・Firestoreスナップショットリスナーを完全削除

## 3. 重要な実装上の注意事項

### プロンプト構造（必ず守ること）
```
system_instruction  → ゲームロジック・JSON スキーマのみ（クリーンな内容）
ユーザーメッセージ冒頭 → [NPC設定: ${char.prompt} 現在の欲情度: ${arousal}%]
```
**エロ系プロンプトを `system_instruction` に入れると空レスポンスになる。絶対にやらないこと。**

### Gemini TTS のクォータ
- モデル: `gemini-2.5-flash-preview-tts`
- 無料枠: **1日100リクエスト**（5〜10回プレイで枯渇）
- 推奨: 設定画面の優先エンジンを **「Google Cloud」** に設定して運用する
- 課金を有効化すれば上限が緩和される

### API キー（両方 Google Cloud Console から取得）
| キー | 用途 |
|---|---|
| `geminiApiKey` | Generative Language API（テキスト生成 + Gemini TTS） |
| `gcpApiKey` | Cloud Text-to-Speech API（Neural2） |

## 5. 開発環境

```bash
npm run dev    # ローカル開発サーバー起動
npm run build  # 本番ビルド → dist/ に出力
git push origin main  # リモートへ反映
```

> 再開時は必ず `localStorage` に有効な Google Cloud Console の API キーが保存されているか確認すること。
