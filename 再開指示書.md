## 2. 2026-04-08 対応内容（コミット 24b302e）

### ① しりとり精度の強化（漢字混じり対応）
- **既出単語チェック**: 漢字混じり入力でも `history` と完全一致 or ひらがな化一致で検出 → 負け処理
- **末尾「ん」チェック**: `isKanaOnly` 判定の外に出し、漢字混じり入力（例: 「ラーメン」「うどン」）でも末尾が `ん/ン` なら検出
- **AI返答の再検証**（`handlePlayerInput` 内の `JSON.parse` 後）:
  1. `result.word` が既出履歴に含まれていれば AI の負けに書き換え
  2. かな入力時のみ、`result.word_reading` の先頭文字がプレイヤー入力の末尾文字と一致するか検証 → 不一致なら1回だけ safeSys リトライ
  3. `result.word_reading` の末尾が `ん` なら AI の負けに書き換え

### ② 欲情度エフェクト強化
- **ピンクグロー**: `arousal > 30` で発動。`@keyframes pulse-pink-glow` による薄ピンクの box-shadow 層
- **吐息テキスト浮遊**: `arousal > 50` で「はぁ…」「んっ…」等が画面下部から浮遊。`arousal > 80` で種類と速度が増す
- **画面呼吸**: `arousal > 60` で背景画像コンテナが `scale(1)↔scale(1.015)` でゆっくり脈動
- 既存の赤 vignette / heart-beat と共存

### ③ APIレート制限の通知UI（トースト）
- `notification` state + `showNotification(text, type, duration)` ヘルパー
- 画面上部にトースト JSX（`warning`=amber / `info`=sky / `error`=red の3色）
- ×ボタンで手動クローズ、デフォルト5秒で自動クローズ
- 呼び出し箇所:
  - Gemini テキスト生成 429 → error トースト（8秒）
  - Gemini TTS 429 → info トースト（GCP切替通知）
  - GCP TTS 429/quota → warning トースト

## 3. 重要な実装上の注意事項

### プロンプト構造（必ず守ること）
```
system_instruction  → ゲームロジック・JSON スキーマのみ（クリーンな内容）
ユーザーメッセージ冒頭 → [NPC設定: ${char.prompt} 現在の欲情度: ${arousal}%]
```
**エロ系プロンプトを `system_instruction` に入れると空レスポンスになる。絶対にやらないこと。**

### Gemini TTS のクォータ
- モデル: `gemini-2.5-flash-preview-tts`
- 無料枠: **1日100リクエスト**（5〜10回プレイで枯渇）
- 推奨: 設定画面の優先エンジンを **「Google Cloud」** に設定して運用する
- 課金を有効化すれば上限が緩和される

### しりとり検証の3層構造
1. **クライアント事前チェック**（`handlePlayerInput` 冒頭）: 既出 / 末尾「ん」 / かな入力時の先頭文字
2. **AI側検証**（`getSystemPrompt` 内のルール）: `valid` / `player_lost` / `sister_lost` フラグ
3. **AI返答事後検証**（`JSON.parse` 後）: 既出 / 読みミスマッチリトライ / 読み末尾「ん」

### API キー（両方 Google Cloud Console から取得）
| キー | 用途 |
|---|---|
| `geminiApiKey` | Generative Language API（テキスト生成 + Gemini TTS） |
| `gcpApiKey` | Cloud Text-to-Speech API（Neural2） |

npm run build  # 本番ビルド → dist/ に出力
