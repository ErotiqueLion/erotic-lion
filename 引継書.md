# プロジェクト引継書：淫らな尻とり (Erotic Word Chain)
**最終更新: 2026-04-05 / コード検証済み**

## 1. プロジェクト概要
音声認識と AI 生成による「しりとり」をベースにした、官能的な要素を含む対話型 Web ゲーム。

- **リポジトリ**: [ErotiqueLion/EWC](https://github.com/ErotiqueLion/EWC.git)
- **技術スタック**: React (Vite) + TailwindCSS + Firebase (Auth/Firestore)
- **主要な API**:
  - **Generative Language API** (Gemini テキスト生成 + Gemini TTS)
  - **Cloud Text-to-Speech API** (Neural2 高品質音声)
  - **Web Speech API** (ブラウザ内蔵、最終フォールバック)

---

## 2. API 設定と認証

## 4. 主要ロジックの詳細 (`src/App.jsx`)

### ゲーム状態遷移
```
locked → intro → character_select / settings / help
                        ↓
                      ready → playing → gameover
```
- `locked`: パスコード入力画面（パスコードは `SECRET_PASSCODE` 定数に定義）
- `playing` → `gameover`: `result.player_lost`、`result.sister_lost`、または `arousal >= 100` で遷移

### 安全フィルター (Safety Filter) とリトライ
`handlePlayerInput` 内で**2段階リトライ**を実装：

1. **1回目**: キャラクター固有のシステムプロンプトで試行
2. **safeSys リトライ**: `finishReason === 'SAFETY'` で null が返った場合、クリーンな `safeSys` に差し替えて再試行

> **現状**: `gemini-2.5-flash` + `BLOCK_NONE` の組み合わせでほぼブロックされない。  
> リトライが頻発する場合はプロンプトの表現見直しを検討。

### 3段階ハイブリッド TTS (`speak` 関数)
設定画面で優先エンジンを選択でき、失敗時は自動フォールバック：
1. **Gemini TTS** → PCM データを `pcmToWav` で変換して再生
2. **GCP TTS** → MP3 形式。欲情度 > 70% 時に `playbackRate` を 1.05 に上げる演出あり
3. **Web Speech API** → 最終フォールバック

優先順位は `localStorage` の `erotic_wordchain_tts_priority` に保存。

### 欲情度システム (`arousal`)
- 0〜100%。AI が返す `arousal_inc`（デフォルト 15）に `arousalMultiplier` を乗算して加算
- `arousalMultiplier` は設定画面のスライダーで 0.5〜3.0 倍に調整可能
- 100% 到達でプレイヤー勝利
- 画像演出: `clothesOpacity = max(0, 1 - arousal/80)`、`blurValue = max(0, 10 - arousal*0.1)`

### `stateRef` パターン
非同期コールバック内で最新の state を参照するため、`stateRef` に state を同期している。  
TTS の `onended` や `callGemini` 内では必ず `stateRef.current` を参照すること。

### `isBusyRef` フラグ
`speak` と `handlePlayerInput` の多重実行を防ぐロック。必ず処理終了時に `false` に戻す。

## 6. Firebase 連携
- **Anonymous Auth** で自動サインイン（`__initial_auth_token` がある場合は Custom Token 認証）
- **Firestore** にゲーム進捗を保存: `artifacts/{appId}/users/{uid}/saveData/current`
  - 保存内容: `arousal`, `displayKana`, `history`, `selectedCharKey`, `timestamp`
- Firebase 未設定（`__firebase_config` が未定義）の場合でも動作するようフォールバック済み

---

## 7. 入力モード
- **マイク入力** (Web Speech API): `navigator.mediaDevices.getUserMedia` で権限取得後に起動
- **テキスト入力**: マイク不可環境で自動切替。ゲーム中にも手動切替ボタンあり
- 入力テキスト末尾に `・・・` を付けて「考え中」を表現（思考中ドットは削除済み）

## 9. 既知の課題・注意事項

- **`aiStartKana` の未使用**: `handlePlayerInput` 内で計算されているが、現在のプロンプトには使われていない（AI 自身が次のかなを判定する設計に変更済み）
- **設定画面の説明文**: Gemini キーの説明に「AI Studio」の記述が残っているが、実際は Cloud Console キーを使用
- **しりとりルール逸脱**: AI が稀にルールを無視する場合がある。`safeSys` の指示強化で対処
- **API レート制限**: `quota` エラー時は待機秒数をメッセージで通知する処理あり
