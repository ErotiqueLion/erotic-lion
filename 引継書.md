# プロジェクト引継書：淫らな尻とり (Erotic Word Chain)

## 1. プロジェクト概要
本作は、音声認識と AI 生成による「しりとり」をベースにした、官能的な要素を含む対話型 Web ゲームです。

- **リポジトリ**: [ErotiqueLion/EWC](https://github.com/ErotiqueLion/EWC.git)
- **技術スタック**: React (Vite) + TailwindCSS + Firebase (Auth/Firestore)
- **主要な API**:
  - **Google Gemini API** (Cloud Generative Language API)
  - **Google Cloud Text-to-Speech API**
  - **Web Speech API** (ブラウザ内蔵の音声認識・合成)

---

## 2. API 設定と認証
本プロジェクトでは、2種類の Google Cloud API キーを使い分けられるように設計されています。

### ① Cloud Generative Language API (Gemini)
- **用途**: 会話テキスト生成 (Gemini 2.5-flash) および Gemini TTS 音声。
- **取得先**: [Google AI Studio](https://aistudio.google.com/app/apikey)
- **注意点**: 2026年現在の正式な REST フィールド名は `system_instruction` (アンダースコア) です。キャメルケースの `systemInstruction` では動作しない場合があります。

### ② Cloud Text-to-Speech API
- **用途**: 最高品質の音声合成 (Journey / Neural2 シリーズ)。
- **取得先**: Google Cloud Console
- **現在の設定**: `ja-JP-Neural2-B` (女性1)、`ja-JP-Neural2-F` (女性2) を主に使用。

---

## 3. 主要ロジックの詳細 (`App.jsx`)

### 安全フィルター (Safety Filter) とリトライ
本作はプロンプトが過激なため、頻繁に Gemini の安全フィルターに接触します。これに対応するため、`handlePlayerInput` 内で**多段リトライロジック**を実装しています。

1.  **通常試行**: 本来の過激なキャラクタープロンプトで実行。
2.  **セーフ・リトライ**: 1回目が空の結果（ブロック）だった場合、自動的に `safeSys` (「あなたは優秀なしりとりAIです」という極めてクリーンな指示) に差し替え、開始文字情報を付与して再試行します。これにより、ゲームのフリーズを防ぎます。

### 3段階ハイブリッドTTS構成
`speak` 関数は以下の優先順位（設定で変更可能）に基づいて音声を再生します。
1.  **Gemini TTS**: `gemini-2.5-flash-preview-tts` モデルを使用. PCMデータを `pcmToWav` で変換して再生。
2.  **GCP TTS**: Cloud Text-to-Speech を使用。MP3形式。
3.  **Web Speech API**: ブラウザ内蔵音声。最終的な安全策。

## 4. UI/UX の工夫
- **インジケーター**: 思考中のドットは削除し、プレイヤーの入力テキスト末尾に `・・・` を付けて「考え中」を表現。
- **状態リセット**: ゲーム開始時やリスタート時に、以前のテロップやテストメッセージをクリアする処理を徹底。
- **画像の変化**: 「欲情度 (arousal)」に応じて、キャラクター画像が `clothed` (通常) から `unveiled` (欲情時) へとフェードし、ブラー（ぼかし）が解除される演出。

## 5. 今後の課題・未解決事項
- **API レート制限 (404/429)**: Gemini の無料枠や GCP TTS のクォータに達した場合の、より親切なユーザー通知。
- **プロンプトの更なる最適化**: 「しりとりルールを守る」ことと「エッチな演技を維持する」ことのバランス調整。たまに AI がルールを無視する場合、`safeSys` の指示をさらに強化する必要があります。

---

> [!IMPORTANT]
> 開発を再開する際は、必ず `localStorage` に有効な Gemini API キーが保存されていることを確認してください。また、`modelVersion` は 2026年現在の主力である `gemini-2.5-flash` を指定しています。

> [!TIP]
> コンソールログの `Raw Gemini Text Response` を監視することで、ブロックの理由やレスポンスの構造をリアルタイムで追跡可能です。
